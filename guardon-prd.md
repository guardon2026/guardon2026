# Product Requirements Document — GuardOn

**Author:** 구본근 대표
**Date:** 2026-04-14

---

## Executive Summary

GuardOn은 대한민국 경비·보안 인력 시장(6,005개 업체, 21.8만 명 종사자, 시장 규모 5~6조 원)에서 **업체 간 인력 공유 B2B 레이어**를 최초로 구축하는 통합 인력 소싱 플랫폼이다. 경비업으로 시작해 행사도우미·안내·미화 등 전체 인력(용역) 시장으로 확장, 3년 내 "없으면 일이 안 되는" 인프라 플랫폼 — 대한민국 인력 매칭 시장 1위 — 을 목표로 한다.

**핵심 문제:** 경호업체가 수주를 받고도 당일 인력 결원이 발생하면 개인 인맥·전화로만 해결한다. 이 문제를 해결하는 디지털 마켓이 존재하지 않는다.

**타겟 사용자:** ① 경비·파견업체 (B2B 인력 수급 주체), ② 경비원·특수인력 구직자 (인력 공급 측), ③ 개인 경호 의뢰 고객 (Growth 단계 추가 예정).

### What Makes This Special

**1. 업체 간 인력 공유 마켓 — 시장에 존재하지 않는 레이어**
알바몬·사람인은 업체↔구직자 연결만 한다. GuardOn은 **업체↔업체** B2B 레이어를 추가한다. A업체의 유휴 인력을 B업체에 즉시 파견하고 수수료를 자동 정산하는 구조로, 현재 어떤 플랫폼도 제공하지 않는 기능이다.

**2. SOS 긴급 매칭 — 핵심 Aha Moment**
수주 후 당일 결원 발생 시 반경 N km 내 가용·검증된 인력에게 즉시 알림 → 수락 → 배치. 전화 돌릴 필요 없음. 이 단 하나의 경험이 업체의 플랫폼 락인을 만든다.

**3. 자격증 인증 뱃지 시스템**
경비지도사·신변보호사·신임경비교육이수·특수경비원 자격증을 플랫폼 내 인증. 업체가 "아는 사람"이 아닌 데이터로 인력을 신뢰할 수 있게 한다.

**4. 초기 네트워크 우위**
G360·KKM 네트워크(파트너 업체 + 훈련 수료생 25,000명)를 인력 DB로 즉시 활용. 콜드스타트 문제를 창업자 고유 자산으로 해결 — 경쟁자가 복제 불가능한 진입 장벽.

**5. 법적 설계 원칙 내재화**
플랫폼 = 직업정보 제공 서비스(직업안정법 제2조). 고용 주체 = 각 경비업체. 불법 파견 논란을 구조적으로 차단.

## Success Criteria

### User Success

경비·파견업체 (B2B 핵심 유저):
- SOS 매칭 요청 후 **5분 이내** 가용 인력 목록 노출
- **SOS 요청의 80% 이상** 당일 인력 배치 확정
- 회원가입 후 **30분 이내** 첫 검색 완료
- 첫 SOS 매칭 성공 업체의 **70% 이상** 1개월 내 재이용

인력 공급자 (구직자):
- 프로필 등록 후 **7일 이내** 첫 매칭 제안 수신
- 자격증 인증 뱃지 발급: 서류 제출 후 **24시간 이내**

## Product Scope

### MVP — Phase 1 (6주, 예산 250만원)

**MVP 접근법:** Experience MVP — SOS 매칭 성공 경험 전달 우선. 매출보다 첫 경험과 네트워크 확보가 목표.
**팀:** 창업자 단독 (Claude Code AI 활용) / **런칭 목표:** G360·KKM 파트너 10개사 베타 → 30개사

Web 전용. 모바일 앱 없음.

| MVP 필수 기능 | 포함 근거 |
|---|---|
| 업체·인력 회원가입 / 경비업 허가번호 인증 | 플랫폼 신뢰 기반 |
| 인력 프로필 등록 (자격증·경력·가용여부·주소) | SOS 매칭 데이터 소스 |
| 자격증 뱃지 4종 (경비지도사·신변보호사·신임경비교육·특수경비원) | 인력 신뢰도 차별화 |
| 인력 검색·필터 (지역·분야·자격증·가용여부) | SOS 전 사전 탐색 경로 |
| **SOS 긴급 매칭**: 등록 → 반경 알림 → 수락 → 확정 | 플랫폼 핵심 가치 |
| 카카오 알림톡 연동 (SOS 알림·확정·취소) | SOS 실시간성 확보 |
| 매칭 이력 조회 | 재사용 유도 |
| 기본 관리자 페이지 (회원 승인·자격증 심사·SOS 모니터) | 운영 최소 조건 |
| 카카오 소셜 로그인 | 가입 마찰 최소화 |

**의도적 MVP 제외:** 업체 간 인력 공유 마켓, 전자계약, 자동 정산, 구독 결제 자동화, 업체 직원 계정, AI 매칭 스코어, 모바일 앱

### Growth — Phase 2 (1~2년차)

업체 간 인력 공유 마켓, 구독 결제 자동화 (월 99,000원), 전자계약·근로계약서 자동 생성 (카카오페이 전자서명), 업체 직원(Staff) 계정, AI 매칭 스코어, 복수 구독 티어 검토, 크라브마가·청원경찰 자격증 추가

### Vision — Phase 3 (3년+)

경비 외 전체 인력(용역) 시장 확장 (행사도우미·안내·미화 등), 모바일 앱 (iOS/Android), 기업 B2B 직접 계약 채널, 자격증 자동 진위확인 API, MOU 기관 채널 (전남경찰청특공대, 인천공항보안)

## User Journeys

### Journey 1. 박성민 대표 — SOS 매칭 성공 (업체, Happy Path)

**페르소나:** 박성민(45세), 서울 소재 경비업체 'S경호' 대표. G360 파트너사. 직원 12명, 연간 수주 30건 이상. 인력 수급을 개인 카카오톡 단체방으로 해결해왔다.

**Opening:** 금요일 오후 2시. 내일 VIP 행사 수주 확정. 필요 인력 8명 중 자사 가용 6명. 나머지 2명을 당장 구해야 한다. 단체방에 올렸지만 응답이 없다.

**Rising Action:** GuardOn 접속 → SOS 버튼 → 장소(강남구)·날짜·필요 인원(2명)·자격증 조건·시급 입력. 30초 만에 등록 완료.

**Climax:** 3분 후 반경 15km 내 가용 인력 7명에게 자동 알림 발송. 4분 후 첫 번째 수락, 6분 후 두 번째 수락. 자격증 뱃지·경력·평점 확인 후 확정. 총 8분. 전화 한 통 없음.

**Resolution:** 박 대표는 그날 저녁 월 99,000원 구독을 결제한다.

**드러난 기능:** SOS 등록 폼, 반경 기반 인력 검색, 카카오 실시간 알림, 인력 프로필 뷰(자격증 뱃지), 매칭 확정 플로우, 이력 저장.

---

### Journey 2. 이재호 — SOS 수락 (경비원·인력 공급측, Happy Path)

**페르소나:** 이재호(32세), 신변보호사 자격증 보유 프리랜서 경비원. KKM 크라브마가 수료생. 고정 업체 없이 지인 소개로 일감을 받아왔다. 수입 불규칙, 좋은 현장 정보 접근 어려움.

**Opening:** 목요일 오후, 다음 주 일정 비어 있음. GuardOn 프로필 가용 상태 ON.

**Rising Action:** 오후 3시 47분, 카카오 알림 도착. "긴급 매칭 — 강남구, 내일 오전 9시, 신변보호사 이상, 시급 18,000원." 업체 프로필·평점 확인.

**Climax:** 수락 버튼 클릭. 즉시 업체 연락처·집결지 정보 수신. 5분 만에 내일 일정 확정.

**Resolution:** 지인 연락 없이 일감이 왔다. 이재호는 가용 상태만 켜두면 일감이 온다는 것을 경험한다.

**드러난 기능:** 가용 상태 토글, 카카오 알림 수신, SOS 요청 상세 뷰, 수락/거절, 확정 후 집결지 공유.

---

### Journey 3. 박성민 대표 — SOS 매칭 실패 (업체, Edge Case)

**상황:** 광주 지방 행사 긴급 수주. 반경 20km 내 가용 인력 전원 기배치. 15분 경과 후 수락 없음.

**플랫폼 대응:**
- 15분 경과 시 반경 자동 50km 확장, 재알림 발송
- "가용 인력 부족 — 반경 확장 또는 조건 조정" 안내 메시지
- 미해결 SOS 건 관리자 대시보드 플래그 표시

**Resolution:** 완전 해결 실패지만 상황을 빠르게 인지해 대안 확보 시간을 확보. 플랫폼 신뢰 유지.

**드러난 기능:** 매칭 실패 상태 표시, 반경 자동 확장 로직, 조건 수정 재시도, 미해결 알림.

---

### Journey 4. 운영 관리자 — 기본 어드민 (MVP)

**페르소나:** GuardOn 운영 담당자 (초기엔 대표 직접 운영).

**핵심 플로우:** 신규 업체 가입 승인 (허가번호 확인 → 수동 승인), 자격증 뱃지 발급 (서류 업로드 → 확인 → 뱃지 부여), 미해결 SOS 모니터링, 기본 통계 조회.

**드러난 기능:** 회원 목록·승인 페이지, 자격증 서류 뷰어, SOS 현황 모니터, 기본 통계 대시보드.

## Domain-Specific Requirements

### Compliance & Regulatory

**직업안정법 제2조 준수**
- 플랫폼 = "직업정보 제공 서비스" (파견업·도급업 아님)
- 이용약관·매칭 계약서에 "플랫폼은 정보 연결, 고용 주체 = 각 경비업체" 명시 필수
- 잡코리아·사람인 등 기존 유사 서비스 법적 구조 참조 적용 가능
- 법무사 선임 및 이용약관 설계: **MVP 출시 전(6주 내) 완료**

**경비업법**
- 업체 가입 시 경비업 허가번호 필수 입력
- 관리자 수동 확인 후 승인 (MVP), 향후 자동 API 검증 검토
- 허가번호 미보유 업체 가입 차단

**개인정보보호법(PIPA)**
- 자격증 정보·위치 정보 수집 시 명시적 동의 별도 수집
- 데이터 보유 기간 정책 수립 (매칭 종료 후 최소 보유 기간 명시)
- 탈퇴 시 개인정보 삭제·익명화 처리

### Technical Constraints

**카카오 알림톡 (SOS 핵심 의존성)**

**위치 기반 반경 검색**
- 인력 거주지 기준 반경 필터링 (실시간 GPS 아님, 프로필 등록 주소 기반)

## Innovation & Novel Patterns

### Detected Innovation Areas

**① 업체 간 인력 공유 B2B 마켓 — 한국 시장 최초**
6,005개 경비업체가 인력 부족·과잉 시 전화·인맥으로만 해결. 이 공백을 채우는 디지털 플랫폼이 국내에 전무. "현재 어떤 플랫폼도 없음"은 마케팅 문구가 아닌 실제 시장 현실.

**② 콜드스타트 문제의 구조적 해결**
마켓플레이스 최대 난관인 닭-달걀 문제를 창업자 고유 자산(G360·KKM 네트워크, 훈련 수료생 25,000명)으로 사전 해결. 대부분의 마켓플레이스가 실패하는 지점을 우회하는 구조적 우위.

**③ 단방향 공고 → 양방향 실시간 매칭 패러다임 전환**
기존 채용 플랫폼: 업체가 공고 올리고 지원자 대기 (수동, 비실시간). GuardOn: SOS 발생 → 플랫폼이 능동적으로 가용 인력에게 알림 발송 → 즉시 수락. 경비업계 최초 Push 기반 매칭 구조.

**④ 업계 특화 신뢰 인프라**
자격증 인증 뱃지 + 경비업 허가번호 게이트. "아는 사람"에서 "데이터"로 신뢰 기반 이동. 일반 플랫폼이 제공할 수 없는 레이어.

### Market Context & Competitive Landscape

- **알바몬·사람인:** 업체↔구직자 연결만. B2B 레이어 없음. 경비 전문성 없음.
- **크몽·숨고:** 프리랜서 마켓. 경비업 허가 검증 없음. 긴급 매칭 구조 없음.
- **GuardOn 포지션:** 두 시장의 교집합에서 독점적 포지션 — 알바몬이 못 하는 B2B 협업 + 크몽이 못 하는 경비 전문성.

### Validation Approach

- **핵심 가설:** "업체는 월 99,000원보다 결원 1건의 손실이 더 크다" → 검증 시 구독 전환 자연 발생
- **MVP 검증 지표:** SOS 매칭 성공률 80% 이상, 업체 재이용율 70% 이상 (첫 1개월)
- **네트워크 효과 검증:** G360·KKM 30개사 온보딩 후 유료 전환율 50% 이상

## SaaS B2B Specific Requirements

### Multi-Tenancy Model

- **테넌트 단위:** 각 경비업체 (업체별 독립 데이터 공간)
- **공개 데이터:** 인력 프로필(자격증·경력·가용여부) — 모든 업체 검색 가능
- **격리 데이터:** 업체별 매칭 이력, 정산 내역, SOS 등록 내역 — 해당 업체만 열람
- **인력 데이터:** 본인 매칭 이력·배치 내역만 열람 가능

### RBAC — 권한 모델

| 역할 | 주요 권한 | MVP |
|---|---|---|
| **업체 대표(Owner)** | SOS 등록·관리, 인력 검색, 매칭 확정, 이력 조회 | ✅ |
| **업체 직원(Staff)** | SOS 등록, 인력 검색 (정산 열람 제한) | Growth |
| **인력(Worker)** | 프로필 관리, 가용 상태 토글, SOS 수락·거절 | ✅ |
| **플랫폼 관리자(Admin)** | 전체 회원 관리, 자격증 승인, SOS 모니터, 통계 | ✅ |

### Subscription Model

- **MVP 단일 티어:** 6개월 무료 베타 → 월 99,000원/업체
- 인력(구직자) 계정: 무료 (매칭 수수료로 수익화)
- 복수 티어: Growth 단계 이후 검토

### Technical Architecture

**데이터 격리:** DB 레벨 Row-level Security (tenant_id 기반) + API 레벨 인증 토큰 검증

**인증:** 카카오 소셜 로그인 + JWT 세션 관리 + 역할별 API 접근 제어

**실시간 알림:** SOS 발생 → 카카오 알림톡 API → 대상 인력 푸시. 실패 시 SMS 폴백 자동 발송. 알림 발송 로그 DB 저장.

**DB 권장:** PostgreSQL (Row-level Security + PostGIS 위치 기반 쿼리)

**MVP 구현 원칙:**
- 반응형 웹 (모바일 브라우저 지원). 네이티브 앱 없음.
- AI 매칭 스코어: MVP는 거리·자격증·가용여부 기반 단순 필터. AI 스코어는 Growth.
- 결제: MVP는 수동 청구 또는 단순 카드 결제. 구독 자동 갱신은 Growth.

## Functional Requirements

### 1. 회원 관리 (User Management)

- FR1: 경비업체는 경비업 허가번호를 입력하여 업체 계정을 생성할 수 있다
- FR2: 경비원·특수인력은 개인 계정을 생성하고 공급자(인력) 역할로 등록할 수 있다
- FR3: 사용자는 카카오 소셜 로그인으로 간편 가입·로그인할 수 있다
- FR4: 업체는 자사 프로필(업체명·소개·주소·연락처·허가번호)을 등록·수정할 수 있다
- FR5: 인력은 개인 프로필(이름·분야·경력·보유 자격증·활동 지역·시급 희망·가용여부)을 등록·수정할 수 있다
- FR6: 인력은 자신의 가용 상태를 ON/OFF 전환할 수 있다
- FR7: 플랫폼 관리자는 신규 업체 가입 신청을 승인 또는 거부할 수 있다

### 2. 자격증 인증 시스템 (Credential Verification)

- FR8: 인력은 자격증 서류(사진·스캔본)를 플랫폼에 업로드하여 인증 신청할 수 있다
- FR9: 플랫폼 관리자는 제출된 자격증 서류를 검토하고 인증 뱃지를 발급하거나 반려할 수 있다
- FR10: 인증된 자격증은 인력 프로필에 공개 뱃지로 표시된다 (경비지도사·신변보호사·신임경비교육이수·특수경비원)
- FR11: 업체는 인력 검색 시 자격증 인증 여부를 필터 조건으로 사용할 수 있다

### 3. 인력 검색 & 탐색 (Workforce Discovery)

- FR12: 업체는 지역(시/구 단위)으로 인력을 검색·필터링할 수 있다
- FR13: 업체는 분야(일반경비·신변보호·특수경비·크라브마가강사 등)로 인력을 필터링할 수 있다
- FR14: 업체는 자격증 종류·가용여부·경력 연수로 인력을 필터링할 수 있다
- FR15: 업체는 검색 결과에서 인력의 프로필(자격증 뱃지·경력·평점·활동 지역)을 조회할 수 있다
- FR16: 인력 검색 결과는 거리·자격증 보유 수·가용여부 기준으로 정렬할 수 있다

### 4. SOS 긴급 매칭 (Emergency Matching)

- FR17: 업체는 SOS 요청을 등록할 수 있다 (장소·날짜·시간·필요 인원·자격증 조건·시급)
- FR18: 시스템은 SOS 등록 즉시 조건에 부합하는 반경 내 가용 인력에게 자동 알림을 발송한다
- FR19: SOS 알림 발송 반경은 기본 20km이며 15분 내 미수락 시 50km로 자동 확장된다
- FR20: 인력은 수신된 SOS 요청을 수락하거나 거절할 수 있다
- FR21: 업체는 수락한 인력의 프로필을 확인하고 최종 확정 또는 거절할 수 있다
- FR22: 매칭 확정 시 업체와 인력 모두에게 확정 알림과 집결지 정보가 전달된다
- FR23: 업체는 진행 중인 SOS 요청의 상태(발송 중·수락 대기·확정·미해결)를 실시간으로 조회할 수 있다
- FR24: 시스템은 SOS 요청이 일정 시간 내 미해결 시 업체에게 미해결 알림을 발송한다

### 5. 알림 & 커뮤니케이션 (Notifications)

- FR25: 시스템은 SOS 관련 이벤트(요청·수락·확정·취소)를 카카오 알림톡으로 발송한다
- FR26: 카카오 알림톡 발송 실패 시 SMS로 자동 폴백 발송된다
- FR27: 사용자는 수신한 알림 이력을 플랫폼 내에서 조회할 수 있다

### 6. 매칭 이력 관리 (Match History)

- FR28: 업체는 자사의 모든 SOS 요청 이력(완료·미해결·취소)을 조회할 수 있다
- FR29: 인력은 자신이 수락·완료한 매칭 이력을 조회할 수 있다
- FR30: 업체는 매칭 완료 후 인력에 대한 평점을 남길 수 있다
- FR31: 인력의 누적 평점은 프로필에 공개되어 신뢰 지표로 활용된다

### 7. 플랫폼 관리 (Administration)

- FR32: 관리자는 전체 회원(업체·인력) 목록을 조회하고 계정을 활성화·비활성화할 수 있다
- FR33: 관리자는 자격증 인증 신청 목록을 조회하고 서류를 검토하여 뱃지를 발급·반려할 수 있다
- FR34: 관리자는 전체 SOS 요청 현황(진행 중·미해결·완료)을 모니터링할 수 있다
- FR35: 관리자는 기본 운영 통계(일별 매칭 건수·신규 가입 수·활성 인력 수)를 조회할 수 있다
- FR36: 관리자는 이상 매칭 건·분쟁 신고를 접수하고 상태를 관리할 수 있다

### 8. 보안 & 규정 준수 (Security & Compliance)

- FR37: 시스템은 회원가입 시 개인정보 수집·이용 동의 및 위치정보 수집 동의를 별도 수집한다
- FR38: 자격증 서류 이미지는 암호화 저장되며 관리자 외 직접 열람이 차단된다
- FR39: 업체의 매칭 이력·정산 데이터는 타 업체에게 노출되지 않는 테넌트 격리 구조로 저장된다
- FR40: 시스템은 경비업 허가번호 미입력 업체의 가입을 차단한다
- FR41: 플랫폼 이용약관에 "플랫폼은 직업정보 제공 서비스이며 고용 주체는 각 업체"임을 명시하고 가입 시 동의를 받는다

## Non-Functional Requirements

### Performance

- NFR-P1: 인력 검색 결과는 요청 후 **2초 이내** 표시되어야 한다
- NFR-P2: SOS 알림은 등록 후 **30초 이내** 대상 인력에게 발송되어야 한다
- NFR-P3: SOS 요청 등록 → 화면 반영까지 **1초 이내** 완료되어야 한다
- NFR-P4: 동시 SOS 요청 **50건**까지 성능 저하 없이 처리되어야 한다 (MVP 기준)

### Reliability

- NFR-R1: 플랫폼 가용성은 **월 99.5% uptime** 이상 유지되어야 한다 (월 최대 다운타임 3.6시간)
- NFR-R2: 카카오 알림톡 발송 실패 시 SMS 폴백이 **60초 이내** 자동 발송되어야 한다
- NFR-R3: 데이터베이스 장애 발생 시 **1시간 이내** 복구되어야 한다
- NFR-R4: 모든 매칭 이벤트(SOS 등록·수락·확정)는 트랜잭션 단위로 처리되어 중복 처리되지 않아야 한다

### Security

- NFR-S1: 모든 데이터는 전송 중 TLS 1.2 이상으로 암호화되어야 한다
- NFR-S2: 자격증 서류 이미지는 AES-256 이상으로 암호화 저장되어야 한다
- NFR-S3: 인증 토큰(JWT)은 만료 시간이 설정되어야 하며 재발급 메커니즘이 구현되어야 한다
- NFR-S4: 업체별 데이터는 Row-level Security로 격리되어야 하며 테넌트 간 데이터 노출이 발생해서는 안 된다
- NFR-S5: 개인정보보호법(PIPA) 준수 — 수집 데이터 최소화, 보유 기간 정책, 탈퇴 시 삭제 처리

### Scalability

- NFR-SC1: MVP 런칭 시 동시 접속자 **200명** 지원, 1년차 **1,000명** 동시 접속 지원 가능한 구조로 설계
- NFR-SC2: 인력 DB **10만 명** 규모에서도 검색 성능(2초 이내)을 유지할 수 있는 인덱싱 구조
- NFR-SC3: 경비 외 타 업종 추가 시 데이터 모델 변경 없이 카테고리 확장 가능한 구조

### Integration

- NFR-I1: 카카오 알림톡 API 장애 시 자동 SMS 폴백 전환 (수동 개입 불필요)
- NFR-I2: 카카오 소셜 로그인 API 장애 시 이메일+비밀번호 로그인 대안 제공
- NFR-I3: 외부 API 연동 응답 시간 **3초 초과** 시 타임아웃 처리 및 사용자 오류 메시지 표시
